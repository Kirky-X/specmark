# Converge — Reconcile tasks with implemented code before archive

An optional step between `apply` (all tasks marked `- [x]`) and `archive`. Converge compares the actual code against `proposal.md` / `design.md` / `tasks.md`, finds gaps where the implementation diverges from or fails to cover the spec, and **appends** missing tasks in a new `## Phase N: Convergence` section of `tasks.md`. Append-only: converge never rewrites or deletes existing tasks.

**Positioning**: After `apply` reports all tasks complete, before `archive`. Use when the change is non-trivial and the implementation may have drifted from the spec (e.g., scope creep, skipped edge cases, decisions made mid-implementation that weren't back-propagated).

**Input**: Optionally specify a change name. If omitted, infer from context or use the Glob tool to list `specmark/changes/*/` directories (subdirectory names are change names). Requires that `tasks.md` has all original tasks marked `- [x]` (otherwise: suggest finishing `apply` first).

**Steps**

1. **Verify apply is complete**

   Read `tasks.md`. If any task outside an existing `## Phase N: Convergence` section is `- [ ]`, pause and suggest: "Apply is incomplete (N tasks open). Finish `/specmark apply` before converge." Do not proceed.

2. **Read all three artifacts and the implemented code**

   Read `proposal.md`, `design.md`, `tasks.md` in full. Then locate and read the code that the tasks touched (file paths from task descriptions + standard source roots). The comparison is code-against-spec, not spec-against-spec.

3. **Run the 4 gap-type passes**

   For each pass, compare code behavior against the spec's stated requirements/design and record gaps.

   | Gap type      | Meaning                                                                  |
   | ------------- | ------------------------------------------------------------------------ |
   | missing       | Spec requires X; code has no implementation of X                         |
   | partial       | Spec requires X; code implements part of X (a sub-case or branch absent) |
   | contradicts   | Spec says X; code does Y (a real contradiction, not just incompleteness) |
   | unrequested   | Code does Z; spec never asked for Z (scope creep / speculative feature)  |

4. **Assign severity to each gap**

   | Severity  | Meaning                                                              |
   | --------- | -------------------------------------------------------------------- |
   | CRITICAL  | Contradicts a core requirement, or missing a security/data-integrity requirement |
   | HIGH      | Missing/partial on a primary functional requirement                 |
   | MEDIUM    | Missing/partial on an edge case or NFR                               |
   | LOW       | Unrequested minor convenience; cosmetic drift                       |

5. **For CRITICAL/HIGH/MEDIUM gaps, append convergence tasks**

   Append a new section to `tasks.md`:

   ```markdown
   ## Phase N: Convergence

   _Generated by /specmark converge on YYYY-MM-DD. Append-only: do not edit prior tasks._

   - [ ] [T###] [P1] [Story?] <gap description> — file: <path>
   - [ ] [T###] [P2] [Story?] <gap description> — file: <path>
   ```

   - Use the **next available T### IDs** (do not renumber existing tasks).
   - Each new task follows the propose.md 5-element format: `- [ ] [T###] [P?] [Story?] Description with file path`.
   - Skip gaps that are LOW and `unrequested` — record them in the convergence narrative (see step 6) but do not create tasks; the user can accept the scope creep or file a follow-up change.
   - `contradicts` gaps always produce a task, regardless of severity (the contradiction must be resolved, not silently accepted).

6. **Write a convergence narrative above the task list**

   Above the new `- [ ]` lines in the `## Phase N: Convergence` section, record a short narrative so archive can tell the story later:

   ```markdown
   ## Phase N: Convergence

   _Generated by /specmark converge on YYYY-MM-DD._

   **Gaps found:** N (CRITICAL: a | HIGH: b | MEDIUM: c | LOW: d)
   **Tasks appended:** M (skipped: K LOW/unrequested, recorded as narrative)
   **Unrequested scope (accepted as-is):** <bulleted list, or "none">

   - [ ] [T###] ...
   ```

7. **Hand off to apply, then archive**

   Prompt: "Convergence appended M tasks. Run `/specmark apply` to close them, then `/specmark archive`."

**Output**

```
## Convergence Complete

**Change:** <name>
**Gaps found:** N (CRITICAL: a | HIGH: b | MEDIUM: c | LOW: d)
**Tasks appended:** M → tasks.md `## Phase N: Convergence`
**Unrequested scope accepted:** <count> (see narrative)

Next: `/specmark apply` (to close appended tasks) → `/specmark archive`
```

**Guardrails**

- **Append-only** — never rewrite, reorder, or delete existing tasks. Convergence tasks live under their own `## Phase N: Convergence` heading with the next-available IDs.
- **All original tasks must be `- [x]`** — converge does not rescue a half-done apply. If tasks remain open, redirect to `apply`.
- **Compare against code, not just artifacts** — reading only the spec and inferring gaps defeats the purpose. The implementation is the source of truth for what was actually done.
- **`contradicts` always produces a task** — even LOW severity contradictions get a task; silent acceptance of contradictions corrupts the spec.
- **`unrequested` LOW does not produce a task** — record in narrative only. Forcing cleanup of every minor scope creep bloats the change.
- **Do not edit proposal.md or design.md** — converge reconciles tasks against code; if the spec itself is wrong, that's a separate propose/exploration decision, not a converge action.
- **Re-runnable** — if apply closes Phase N tasks and new drift appears, converge can append `## Phase N+1: Convergence`. Each run is a new phase.

**Fluid Workflow Integration**

- Sits between `apply` (done) and `archive`. May be skipped for trivial changes.
- Pairs with `analyze`: analyze catches spec↔tasks drift pre-apply; converge catches tasks↔code drift post-apply. Together they cover both directions.
- After converge appends tasks, control returns to `apply` to close them; `archive` should only run when all phases (including convergence phases) are `- [x]`.
